Canny Edge Detector

Implementation

I implemented the Canny Edge Detector in five steps:

  1. Convert the color image to an intensity image.
    Using the formula given in class, each pixel's intensity is:
    I(x,y)=O(x,y,B)*0.1+O(x,y,R)*0.3+O(x,y,G)*0.6 where O is the original image and the third dimension is the color channel. You can see the implementation in _intensify starting at line 767.
  2. Compute the gradiant
    This step is a Gaussian blur of the original image followed by a computation of the derivative of the blurred image. Although not necessary for edge detection, my implementation of gradient computation uses the partial derivatives of the image in the X and Y dimensions.
    I calculate partial derivatives in the X and Y dimensions by combining the Gaussian blur with the gradient calculation. Therefore, these two sub-steps happen in one loop through the image. I calculate the combined convolutions in 1-dimension as the derivative of the continuous Gaussian function rather than as an approximation using finite differences. In other words,
    ∂∂XG(x,σ)=−x1σ 2G(x,σ) I combine the two 1-dimensional derivatives to create the convolution matrix and store the derivatives separately. You can see the implementation in _compute_separate_gradient starting on line 901.
    In edge detection I only need the magnitude of the gradient and its direction. To find such values, I combine the partial derivatives by calculating the square root of the sum of the squares of the two components. See line 897 where I am computing the combined gradient using the separate gradients (_compute_gradient).
  3. Thin the gradiant
    I simply followed the steps outlined in the assignment. See _thin_gradient starting on line 793.
  4. "Relativize" the thinned gradiant
    This step brightened the strongest edges to full intensity and then rescales all other edges. This makes the edges in the image easier to see.
  5. Calculate connected edges I calculate connected edges based on a start and continue threshold. Thanks to an idea from a fellow classmate, I decided to specify these thresholds with respect to the mean edge intensity (after they were relativized in the previous step). The thresholds are in [0,1]. See _connected_edges on line 614.
Although there are built-in Gaussian filters supplied by numpy/skimage, I took the opportunity to code everything by hand. The resulting implementation is definitely much slower than the "native" version, but I have mastered the material by not relying on those functions.
There are places where I have improved on my initial naive implementation. There are hotspots in the code that I detected using Python profilers. You can see an example of an optimization at line 953.

Harris Corner Detector

Implementation

  1. Covariance computation at each point
    For each (x,y) in the image,
    • Calculate the covariance matrix around that point
      Because I implemented the gradient calculation needed for Canny in separate dimensions, computing the covariance matrix is a straightforward invocation of that functionality. See line 667.
    • Compute the Eigenvalue of the covariance matrix
      I use the built-in eig() from numpy.linalg to quickly compute this value.
    • Compare the smaller eigenvalue with a threshold
      If the smaller of the two eigenvalues exceeds a threshold, I add (x,y) to a set of candidate corner points. See line 701.
  2. Local maximum amplification
    • Sort the candidate corner points in descending order according to their eigenvalues.
      See line 719.
    • For each candidate corner point, clear nearby candidate corner points if their eigenvalues are smaller.
      See line 721.
  3. Brighten the corners. I use the same relativizing technique as above to make it easier to see corners. See line 736.

Scale Invariant Feature Transform (SIFT)

As Dr. Lowe says in [Lowe 2004], SIFT is a cascading filter. The computationally intensive criteria are only applied to points that meet initial criteria. The initial candidate points come from the calculation of the difference of Gaussians (DoGs). This is described in [Lowe 2004] in Section 3. Those candidate points are filtered based on their location relative to an extremum of a 3-dimensional function generated by fitting the DoG to the candidate. This is described in [Lowe 2004] in Section 4.
In my implementation, candidate selection and filtering are intertwined as the DoGs in each octave are calculated. However, I will describe my implementation as if they are two separate stages.

Selection of Candidate Extrema (Section 3)

The DoG function should be a continuous function in three variables: x, y and σ. However, calculating this function is expensive and the values of σ would have to be arbitrarily close together to be able to find all the extrema (as Dr. Lowe points out in Section 3.1).
Based on empirical results (Section 3.1), Dr. Lowe calculates a value for the difference between σs that generates stable keypoints without too much overhead. He calls this value k.
He also presents an efficient way to calculate the DoG based on octaves and scales. An octave is a doubling of the value of σ. Within each octave, the initial image is scaled s+3* times. Each scale is blurred by k sσ. For example, consider k=2 13 and s=3.

s=5k 3σ2 53σ
s=4k 3σ2 43σ
s=3k 3σ2 33σ=2σ
s=2k 2σ2 23σ
s=1k 1σ2 13σ
s=0k 0σσ
Those scales are subtracted from one another to calculate the DoGs. To make sure that the entire range of σ are covered for the DoGs, multiple octaves are calculated. To move between octaves, the previous octave's 2σ image is downsampled by 2. That image is used as the starting image for the next octave and the process repeats. Because of the selection of the image and the downsampling, the base image for the new octave has, essentially, an embedded 4σ.
My implementation of calculation of the scales of an octave and the DoGs is in _octave and _dog.
To detect extrema, each point (x,y) at each scale σ is compared to its 26 neighbors. Those neighbors are defined as the points in the nearest 3x3 locations at the current scale, the previous scale and the next scale. If the value of the DoG at (x,y,σ) is either smallest or largest, that point is a candidate keypoint location.
My implementation of extrema calculation is in _scale_space_extrema.

Derivation and Implementation of Filtering (Section 4)

In Section 4 of [Lowe 2004], Dr. Lowe presents a technique for subpixel localization that is used to filter out unstable candidate keypoints. The technique is based on the Taylor Series expansion of the DoG evaluated at the candidate keypoints. Here is the Taylor Series expansion to the second quadratic term: f(x)=f(a)+f′(a)1!(x−a)+f″(a)2!(x−a) 2 Taking the derivative of f(x): f′(x)=f′(a)+12f″(a)2x−12f″(a)2a f′(x)=f′(a)+xf″(a)−af″(a) f′(x)=f′(a)+f″(a)(x−a) and therefore x−a=−f″(a) −1f′(a) This is the equivalent of Dr. Lowe's equation (3) when f(a) is the value of the DoG at the candidate location a.
Solving for x−a is possible, as Dr. Lowe points out, using Hessian matrixes and approximating the derivatives using finite differences.
Thanks to [1], I was able to determine the Hessian used to take the second derivative of the DoG: [∂ 2D∂σσ ∂ 2D∂σy ∂ 2D∂σx ∂ 2D∂σy ∂ 2D∂yy ∂ 2D∂yx ∂ 2D∂σx ∂ 2D∂yx ∂ 2D∂xx] To solve for x−a, solve the linear system of equations: [∂ 2D∂σσ ∂ 2D∂σy ∂ 2D∂σx ∂ 2D∂σy ∂ 2D∂yy ∂ 2D∂yx ∂ 2D∂σx ∂ 2D∂yx ∂ 2D∂xx][σ y x]=−[∂D∂σ ∂D∂y ∂D∂x] Implementing the solver was a battle with details. You can see my implementation starting on line 266. On my first attempt, I was able to calculate the differences and move the candidate point. However, this did not seem to converge reliably. Taking a hint from a well-known reference implementation [2], I limited the number of attempted interpolations to 5.** After applying this method of detecting "flapping" between nearby points, the filtering process started to work!
The result of this process is a point in three-dimensional space that indicates where the keypoint should be if it were a true extremum of the fitted function. Based on the difference between the value of the DoG at the ideal location and the value of the DoG at the actual location, the candidate keypoints are filtered. Based on empirical results, Dr. Lowe suggests a difference of 0.03 for filtering. You can see my implementation of this starting on line 343.
